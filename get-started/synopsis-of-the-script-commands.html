由於您可以使用指令碼命令來定義及管理儲存陣列的不同層面（例如主機拓撲、磁碟機組態、控制器組態、磁碟區定義及磁碟區群組定義）、因此實際的命令數量相當龐大。不過、這些命令屬於一般類別、當您套用命令來設定或維護儲存陣列時、就會重複使用這些命令。下表列出指令碼命令的一般格式及每個命令的定義。

語法 說明 
accept object
{statement-data}

執行擱置的作業。

 
activate object
{statement-data}

設定環境、以便在環境已正確設定時執行作業或執行作業。

 
autoConfigure storageArray
{statement-data}

自動建立以命令中指定參數為基礎的組態。

 
check object
{statement-data}

啟動作業以報告物件中的錯誤、這是同步作業。

 
clear object
{statement-data}

捨棄物件部分屬性的內容。此作業具有破壞性、無法回復。

 
create object
{statement-data}

建立指定類型的物件。

 
deactivate object
{statement-data}

移除作業環境。

 
delete object

刪除先前建立的物件。

 
diagnose object
{statement-data}

執行測試並顯示結果。

 
disable object {statement-data}

防止功能運作。

 
download object
{statement-data}

將資料傳輸至儲存陣列或與儲存陣列相關的硬體。

 
enable object
{statement-data}

設定要操作的功能。

 
load object
{statement-data}

將資料傳輸至儲存陣列或與儲存陣列相關的硬體。此命令在功能上與「download'」命令類似。

 
recopy object
{statement-data}

使用現有的Volume複製配對重新啟動Volume複製作業。您可以在重新啟動作業之前變更參數。

 
recover object
{statement-data}

從儲存的組態資料和陳述式參數重新建立物件。（此命令類似於「create」命令。）

 
remove object
{statement-data}

移除物件之間的關聯。

 
repair object
{statement-data}

修復由「Check」命令所發現的錯誤。

 
replace object
{statement-data}

指定的物件會取代儲存陣列中的現有物件。

 
reset object
{statement-data}

將硬體或物件恢復為初始狀態。

 
resume object

啟動暫停的作業。此作業會在暫停時從離開處開始。

 
revive object

強制物件從「失敗」狀態進入「最佳」狀態。此命令只能作為錯誤恢復程序的一部分使用。

 
save object
{statement-data}

將物件的相關資訊寫入檔案。

 
set object
{statement-data}

變更物件屬性。所有變更都會在命令傳回時完成。

 
show object
{statement-data}

顯示物件的相關資訊。

 
start object
{statement-data}

啟動非同步作業。您可以在某些作業啟動後停止。您可以查詢某些作業的進度。

 
stop object
{statement-data}

停止非同步作業。

 
suspend object
{statement-data}

停止作業。然後您可以重新啟動暫停的作業、並從暫停點繼續執行。

 
validate object
{statement-data}

驗證安全金鑰。
